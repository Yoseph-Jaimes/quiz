Tipos de Relaciones JPA

Cómo se mapean las relaciones en tu modelo de datos

@OneToOne
Club → Entrenador

Un club tiene un solo entrenador. La FK está en la tabla club.

@JoinColumn(name = "entrenador_id")
1→N
@OneToMany
Club → Jugadores

Un club tiene muchos jugadores. Con @JoinColumn evita tabla intermedia.

@JoinColumn(name = "club_id")
N→1
@ManyToOne
Club → Asociación

Muchos clubes pertenecen a una asociación. FK en la tabla club.

@JoinColumn(name = "asociacion_id")
N↔N
@ManyToMany
Club ⇄ Competiciones

Muchos clubes participan en muchas competiciones. Tabla intermedia.

@JoinTable(name = "club_competiciones")

FetchType: ¿Lazy o Eager?

En este proyecto usamos FetchType.LAZY para todas las relaciones, mejorando el rendimiento y evitando el problema N+1. Solo se cargan los datos cuando realmente se necesitan.

Las relaciones @OneToOne, @OneToMany, @ManyToOne y @ManyToMany están configuradas con fetch = FetchType.LAZY.
